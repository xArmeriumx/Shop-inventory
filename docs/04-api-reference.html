1. Overview

1.1 Architecture

All business logic is implemented as Next.js Server Actions (the 'use server' directive). These functions run server-side and are called directly from React components. There is no REST or GraphQL API layer.

1.2 Common Patterns

PatternDescription
RBAC GuardEvery action calls requirePermission('PERMISSION_NAME') as its first statement. This authenticates the user, checks their role, and returns a context object (ctx) containing userId, shopId, and role.
Multi-Tenant IsolationAll database queries include shopId: ctx.shopId in the where clause. Data from other shops is unreachable.
Input ValidationInputs are validated using Zod schemas. Invalid inputs return { success: false, error: { fieldname: [...] } }.
Soft DeleteDelete operations set deletedAt instead of removing the row. Queries filter by deletedAt: null.
Optimistic LockingUpdate operations check version to prevent concurrent modification conflicts.
RevalidationAfter mutations, revalidatePath() is called to refresh Next.js caches for affected routes.
Financial PrecisionAll monetary values use Decimal.js internally and are converted with toNumber() before returning to the client.

1.3 Response Format

// Success
{ success: true, data: T }

// Error (validation)
{ success: false, error: { fieldName: string[] } }

// Error (general)
{ success: false, error: { _form: string[] } }

2. Sales sales.ts

FunctionPermissionDescription
getSales(params)SALE_VIEWGet paginated list of sales. Supports search, date range, payment method, channel, and status filters.
getSale(id)SALE_VIEWGet single sale with items, customer, and product details.
createSale(input)SALE_CREATECreate a new sale. Runs inside a transaction: validates products, calculates discounts, reserves stock atomically, generates invoice number with retry logic. Uses Decimal.js for all financial math.
cancelSale(input)SALE_DELETECancel a sale. Restores stock atomically, sets sale status to CANCELLED, cancels pending shipments.
getTodaySales()DASHBOARD_VIEWAggregate today's sales count and revenue.
getRecentSales(limit)DASHBOARD_VIEWGet the N most recent sales for dashboard display.
verifyPayment(saleId, status, note?)PAYMENT_VERIFYApprove or reject a transfer payment. Sets paymentStatus to VERIFIED or REJECTED.
uploadPaymentProof(saleId, proofUrl)SALE_VIEWAttach a payment slip image URL to a sale.

2.1 createSale — Input Schema (SaleInput)

FieldTypeRequiredValidation
itemsArrayYesMin 1 item. Each: productId, quantity (int, min 1), discount (0-100%)
paymentMethodEnumYesCASH | TRANSFER | CREDIT
channelStringNoSales channel identifier
customerIdStringNoExisting customer UUID
noteStringNoMax 500 characters
discountTypeEnumNoPERCENT | FIXED
discountValueNumberNoMust be >= 0. If PERCENT, max 100.
proofOfPaymentStringNoURL to payment slip image

2.2 createSale — Side Effects

  • Generates invoice number (INV-00001) with collision-retry (max 5 attempts)
  • Atomically decrements stock for each item (executeRaw UPDATE ... SET stock = stock - @quantity WHERE stock >= @quantity)
  • Records profit per item: revenue - (costPrice * quantity)
  • Revalidates paths: /sales, /products, /dashboard

2.3 cancelSale — Side Effects

  • Atomically restores stock for each item
  • Cancels any associated shipments with status PENDING or SHIPPED
  • Records cancellation reason and detail
  • Revalidates: /sales, /products, /dashboard, /shipments

4. Purchases purchases.ts

FunctionPermissionDescription

cancelPurchase(input)PURCHASE_DELETECancel with reason. Checks stock sufficiency before reversal. Reverts cost prices.

4.1 createPurchase — Side Effects

  • Generates purchase number (PUR-00001) with collision retry
  • Increments stock for each purchased item
  • Updates each product's costPrice to the new purchase cost
  • Revalidates: /purchases, /products, /dashboard

5.1 createReturn — Input (CreateReturnInput)

FieldTypeValidation
saleIdStringRequired. UUID of parent sale.
reasonStringRequired. Max 500 characters.
refundMethodEnumCASH | TRANSFER | CREDIT
itemsArrayMin 1. Each: saleItemId, quantity (int, min 1), refundPerUnit (min 0).

5.2 createReturn — Side Effects

  • Generates return number (RET-00001) with collision retry
  • Restores stock via StockService
  • Updates parent sale item's returnedQuantity
  • Adjusts parent sale's netAmount and profit
