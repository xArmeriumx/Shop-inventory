2. Environment Variables

Copy .env.example to .env and fill in values:

VariableRequiredDescription
DATABASE_URLYesPostgreSQL connection string. For Supabase: use the pooled connection URL with ?pgbouncer=true&connect_timeout=30.
AUTH_SECRETYesNextAuth.js secret. Generate with openssl rand -base64 32.
AUTH_URLYesFull app URL. http://localhost:3000 for local, production domain for deploy.
NEXT_PUBLIC_SUPABASE_URLYesSupabase project URL (from dashboard → Settings → API).
SUPABASE_SERVICE_ROLE_KEYYesSupabase service role key (server-side only, bypasses RLS).
NEXT_PUBLIC_SUPABASE_ANON_KEYYesSupabase anon key (for client-side storage uploads).
GROQ_API_KEYNoGroq API key. Only required for AI receipt scanner feature.

⚠️ Security Warning: Never commit .env to Git. The .gitignore already excludes it. For production, use your hosting provider's secret management (Vercel Environment Variables, etc.).

3. Local Development Setup

Step 1: Clone and Install

git clone https://github.com/your-org/shop-inventory.git
cd shop-inventory
npm install

postinstall automatically runs prisma generate to create the Prisma client.

Step 2: Configure Environment

cp .env.example .env
# Edit .env with your database credentials and keys

Step 3: Setup Database

# Push schema to database (creates tables)
npm run db:push

# Seed initial data (optional)
npm run db:seed

Step 4: Start Development Server

npm run dev
# App runs at http://localhost:3000

4. Database Setup

4.1 Supabase (Recommended)

  1. Create a new project at supabase.com
  2. Go to Settings → Database and copy the Connection string (URI)
  3. Use the pooled (port 6543) connection for the app: add ?pgbouncer=true&connect_timeout=30
  4. Set DATABASE_URL in your .env
  5. Run npm run db:push to create all tables

4.2 Self-Hosted PostgreSQL

# Create database
createdb shop_inventory

# Set connection string
DATABASE_URL="postgresql://user:pass@localhost:5432/shop_inventory"

# Push schema
npm run db:push

4.3 Schema Management

This project uses Prisma db push (not migrations) for schema management. Changes to prisma/schema.prisma are pushed directly:

# After editing schema.prisma
npm run db:push      # Apply changes to database
npm run db:generate  # Regenerate TypeScript types

5. Supabase Storage

5.1 Required Buckets

Create these two storage buckets in the Supabase dashboard (Storage → New bucket):

Bucket NamePublicPurpose
receiptsYesPayment slips and receipt images
productsYesProduct images

5.2 RLS Policies

Because the app uses the Service Role Key for server-side uploads, RLS is bypassed. However, for public read access, ensure the SELECT policy allows anonymous reads on both buckets.

6. Production Deployment

6.1 Vercel (Recommended)

  1. Connect your Git repository to Vercel
  2. Set all environment variables in Vercel dashboard (Settings → Environment Variables)
  3. Vercel auto-detects Next.js and configures build settings
  4. Deploy triggers automatically on push to main

6.2 Build Settings

SettingValue
FrameworkNext.js
Build Commandnpm run build
Output Directory.next
Install Commandnpm install
Node.js Version20.x

6.3 Custom Server / VPS

# Build
npm ci --production=false
npm run build

# Start with PM2
pm2 start npm --name "shop-inventory" -- start

# Or with Docker
docker build -t shop-inventory .
docker run -p 3000:3000 --env-file .env shop-inventory

6.4 Post-Deploy Checklist

  • ☐ All environment variables set
  • ☐ npm run db:push executed against production database
  • ☐ Supabase storage buckets created with correct policies
  • ☐ AUTH_URL set to production domain (e.g. https://shop.example.com)
  • ☐ HTTPS enabled (automatic on Vercel, configure on VPS)
  • ☐ First user registered and shop created

8. Monitoring and Logging

8.1 Built-in SystemLog

The app logs errors to the SystemLog database table with structured fields: level, message, stack trace, request path, user context. View these in Prisma Studio:

npm run db:studio
# Navigate to the SystemLog table

8.2 Health Check

Call the getSystemHealth() server action to verify database connectivity.

9. Backup and Recovery

9.1 Database Backups

  • Supabase: Automatic daily backups (Pro plan) or use pg_dump manually
  • Self-hosted: Schedule pg_dump via cron
# Manual backup
pg_dump $DATABASE_URL > backup_$(date +%Y%m%d).sql

# Restore
psql $DATABASE_URL < backup_20260209.sql

9.2 Storage Backups

Use the Supabase CLI or API to periodically download all files from the receipts and products buckets.

10. Troubleshooting

Common Issues

ProblemCauseSolution
Build fails: "Prisma Client not generated"Missing postinstallRun npx prisma generate
Database connection timeoutMissing pgbouncer paramAdd ?pgbouncer=true&connect_timeout=30 to DATABASE_URL
Upload fails: "Missing Supabase env"Missing env varsSet NEXT_PUBLIC_SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY
"Invalid AUTH_SECRET"Secret too short or missingRun openssl rand -base64 32 and set AUTH_SECRET
Styles broken in productionTailwind purge issueEnsure content in tailwind.config includes all component paths
AI scanner not workingMissing GROQ_API_KEYSet GROQ_API_KEY in .env. Feature is optional.
Permission denied errorsRBAC misconfiguredCheck user's role has required permissions. Use the Role Management page in Settings.
Stock goes negativeShould not happenAtomic stock operations prevent this. If seen, check for direct DB edits bypassing the app.
