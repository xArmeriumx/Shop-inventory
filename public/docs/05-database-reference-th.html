3. ผู้ใช้ (User)

ฟิลด์ประเภทหมายเหตุ
idString @idcuid()
nameStringชื่อผู้ใช้
emailString @uniqueอีเมล (ไม่ซ้ำ)
passwordStringbcrypt hash
imageString?URL รูปโปรไฟล์

ความสัมพันธ์: มีหลาย Shops (เจ้าของ), ShopMembers, Products, Sales, Purchases, Expenses, Incomes, StockLogs, Shipments, Returns

4. ร้านค้าและ RBAC

Shop (ร้านค้า)

ฟิลด์ประเภทหมายเหตุ
nameStringชื่อร้าน
address / phone / taxIdString?ข้อมูลร้าน
invoicePrefixStringคำนำหน้าใบเสร็จ (เริ่มต้น "INV")
ownerIdStringFK → User (ผู้สร้าง)

Role (บทบาท)

ฟิลด์: name, description, permissions Permission[], isDefault, shopId

ShopMember (สมาชิก)

ฟิลด์: userId, shopId, roleId, type ("OWNER"/"MEMBER"), permissionVersion

Unique: @@unique([userId, shopId])

5. สินค้าและซัพพลายเออร์

Product (สินค้า)

ฟิลด์ประเภทหมายเหตุ
nameStringชื่อสินค้า
skuString?Unique ต่อร้าน
category / categoryIdString / String?หมวดหมู่ (legacy + FK → LookupValue)
costPrice / salePriceDecimal(10,2)ราคาต้นทุน / ราคาขาย
stockIntจำนวนสต็อกปัจจุบัน
minStockIntสต็อกต่ำสุด (ใช้แจ้งเตือน)
isLowStockBooleancomputed flag
versionIntOptimistic Locking
supplierIdString?FK → Supplier

Supplier (ซัพพลายเออร์)

ฟิลด์: name, code, contactName, phone, email, address, taxId, notes

6. การขาย

Sale (รายการขาย)

ฟิลด์หลัก: invoiceNumber (unique ต่อร้าน), totalAmount, totalCost, profit, netAmount, discountType/Value/Amount, paymentMethod, paymentStatus, channel, status, cancelledAt/By/Reason

ดัชนีรวม: [shopId, date, status], [shopId, paymentStatus], [shopId, channel]

SaleItem (รายการสินค้าในบิล)

ฟิลด์: saleId, productId, quantity, salePrice, costPrice, subtotal, profit, discountAmount

7. การซื้อ

Purchase / PurchaseItem

โครงสร้างคล้าย Sale แต่ไม่มี discount, payment status, channel — มี purchaseNumber, supplierId, totalAmount, status

8. คืนสินค้า

Return

ฟิลด์: returnNumber (unique ต่อร้าน), saleId, reason, refundAmount, refundMethod, status

ReturnItem

ฟิลด์: returnId, saleItemId, productId, quantity, refundPerUnit, refundAmount

9. จัดส่งและที่อยู่

Shipment (การจัดส่ง)

ฟิลด์: shipmentNumber, saleId, recipientName/Phone, shippingAddress, trackingNumber, shippingProvider, shippingCost, status (ShipmentStatus enum), shippedAt, deliveredAt

CustomerAddress (ที่อยู่ลูกค้า)

ฟิลด์: customerId, label, recipientName, phone, address, district, subDistrict, province, postalCode, isDefault

10. รายจ่ายและรายรับ

Expense / Income

โครงสร้างเหมือนกัน: category (legacy), categoryId (FK → LookupValue), amount, description, date, receiptUrl. Income มี notes เพิ่ม

ดัชนีรวม: [shopId, date], [shopId, category, date]

11. สต็อกและ Lookups

StockLog (บันทึกการเคลื่อนไหวสต็อก)

ฟิลด์: type (StockMovementType), productId, quantity, balance (snapshot), saleId, purchaseId, returnId, note, date

LookupType / LookupValue

ระบบ dropdown ปรับแต่งได้: หมวดหมู่สินค้า, หมวดหมู่ค่าใช้จ่าย, หมวดหมู่รายรับ. LookupValue มี code, name, color, sortOrder, isActive, isDefault

12. ระบบและแจ้งเตือน

SystemLog

ฟิลด์: level (LogLevel), message, stack, path, method, body, userId, shopId

Notification (การแจ้งเตือน)

ฟิลด์: shopId, userId (null = ส่งทุกคน), type (LOW_STOCK, NEW_SALE ฯลฯ), severity, title, message, link, metadata (Json), isRead, groupKey (dedup), expiresAt

Customer (ลูกค้า)

ฟิลด์: name, phone, email, address, taxId, notes. มีหลาย Sales, CustomerAddresses
