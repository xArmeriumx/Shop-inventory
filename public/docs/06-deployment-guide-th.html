2. Environment Variables

คัดลอก .env.example เป็น .env แล้วกรอกค่า:

ตัวแปรจำเป็นคำอธิบาย
DATABASE_URLใช่Connection string PostgreSQL. สำหรับ Supabase ใช้ pooled URL พร้อม ?pgbouncer=true
AUTH_SECRETใช่NextAuth secret — สร้างด้วย openssl rand -base64 32
AUTH_URLใช่URL แอป เช่น http://localhost:3000 หรือ domain production
NEXT_PUBLIC_SUPABASE_URLใช่Supabase project URL
SUPABASE_SERVICE_ROLE_KEYใช่Service role key (server-side only)
NEXT_PUBLIC_SUPABASE_ANON_KEYใช่Anon key (client-side uploads)
GROQ_API_KEYไม่สำหรับ AI สแกนใบเสร็จเท่านั้น

⚠️ คำเตือน: อย่า commit ไฟล์ .env ลง Git — ใช้ secret management ของ hosting (เช่น Vercel Environment Variables)

3. ตั้งค่า Development

ขั้นที่ 1: Clone และ Install

git clone https://github.com/your-org/shop-inventory.git
cd shop-inventory
npm install     # postinstall จะรัน prisma generate อัตโนมัติ

ขั้นที่ 2: ตั้งค่า Environment

cp .env.example .env
# แก้ไข .env ด้วยข้อมูลจริงของคุณ

ขั้นที่ 3: ตั้งค่าฐานข้อมูล

npm run db:push    # สร้างตารางทั้งหมด
npm run db:seed    # ใส่ข้อมูลเริ่มต้น (เลือกได้)

ขั้นที่ 4: เริ่มเซิร์ฟเวอร์

npm run dev        # แอปรันที่ http://localhost:3000

คำสั่งที่มี

คำสั่งหน้าที่
npm run devเริ่ม dev server พร้อม hot reload
npm run buildBuild production bundle
npm run startเริ่ม production server
npm run lintตรวจสอบ ESLint
npm run testรัน Vitest (watch mode)
npm run db:pushPush schema ไปฐานข้อมูล
npm run db:studioเปิด Prisma Studio (GUI)
npm run db:seedใส่ข้อมูลตัวอย่าง

4. ตั้งค่าฐานข้อมูล

4.1 Supabase (แนะนำ)

  1. สร้างโปรเจกต์ที่ supabase.com
  2. ไปที่ Settings → Database → คัดลอก Connection string (URI)
  3. ใช้ pooled connection (port 6543) เพิ่ม ?pgbouncer=true&connect_timeout=30
  4. ตั้งค่า DATABASE_URL ใน .env
  5. รัน npm run db:push

4.2 จัดการ Schema

โปรเจกต์นี้ใช้ Prisma db push (ไม่ใช้ migration) เมื่อแก้ไข prisma/schema.prisma:

npm run db:push      # Apply การเปลี่ยนแปลง
npm run db:generate  # สร้าง TypeScript types ใหม่

6. Deploy Production

6.1 Vercel (แนะนำ)

  1. เชื่อมต่อ Git repository กับ Vercel
  2. ตั้งค่า environment variables ทั้งหมดใน Vercel dashboard
  3. Vercel ตรวจจับ Next.js อัตโนมัติ
  4. Deploy อัตโนมัติเมื่อ push ไปที่ main

6.2 VPS / Custom Server

# Build
npm ci --production=false
npm run build

# รันด้วย PM2
pm2 start npm --name "shop-inventory" -- start

6.3 Checklist หลัง Deploy

  • ☐ Environment variables ตั้งครบ
  • ☐ npm run db:push รันกับ production database แล้ว
  • ☐ Supabase storage buckets สร้างแล้ว
  • ☐ AUTH_URL ตั้งเป็น production domain
  • ☐ HTTPS เปิดใช้งาน
  • ☐ สมัครผู้ใช้แรกและสร้างร้านค้า

7. Monitoring

  • SystemLog: ระบบบันทึก error ลงตาราง SystemLog — ดูผ่าน Prisma Studio
  • Health Check: เรียก getSystemHealth() ตรวจสอบการเชื่อมต่อ DB
  • แนะนำ: Vercel Analytics (Web Vitals), Supabase Dashboard (DB metrics), Sentry (Error tracking)

8. Backup

  • Supabase: backup อัตโนมัติรายวัน (Pro plan) หรือใช้ pg_dump
  • Storage: ใช้ Supabase CLI ดาวน์โหลดไฟล์จาก buckets เป็นระยะ
# backup ด้วยมือ
pg_dump $DATABASE_URL > backup_$(date +%Y%m%d).sql

# restore
psql $DATABASE_URL < backup_20260209.sql

9. แก้ไขปัญหา

ปัญหาสาเหตุวิธีแก้
Build fail: "Prisma Client not generated"ไม่ได้รัน generateรัน npx prisma generate
Database connection timeoutขาด pgbouncer paramเพิ่ม ?pgbouncer=true&connect_timeout=30
Upload fail: "Missing Supabase env"ไม่ได้ตั้งค่า envตั้งค่า NEXT_PUBLIC_SUPABASE_URL และ SUPABASE_SERVICE_ROLE_KEY
"Invalid AUTH_SECRET"Secret สั้นเกินไปสร้างด้วย openssl rand -base64 32
AI scanner ไม่ทำงานไม่มี GROQ_API_KEYตั้ง GROQ_API_KEY ใน .env (ฟีเจอร์เลือกได้)
Permission deniedRBAC ไม่ถูกต้องตรวจสอบบทบาท/สิทธิ์ของผู้ใช้ในหน้าตั้งค่า
