1. ภาพรวมและ Patterns

สถาปัตยกรรม

ระบบใช้ Next.js Server Actions (ไม่มี REST API) ทุกฟังก์ชันเรียกจาก client ผ่าน 'use server'

Patterns หลัก

Patternรายละเอียด
RBACทุกฟังก์ชันเรียก requirePermission() ตรวจสอบสิทธิ์ก่อนทำงาน
Multi-tenancyshopId ทุก query — แยกข้อมูลระหว่างร้าน
Input ValidationZod schemas ตรวจสอบข้อมูลขาเข้า
Soft DeletedeletedAt: null กรองข้อมูลที่ถูกลบ
Optimistic Lockingตรวจสอบ version ก่อนอัปเดตสินค้า
RevalidationrevalidatePath() หลัง mutation ทุกครั้ง
Financial PrecisionDecimal.js + toNumber() ป้องกัน floating point error

2. สินค้า (Products)

ไฟล์: src/actions/products.ts

ฟังก์ชันสิทธิ์รายละเอียด
getProducts()PRODUCT_VIEWรายการสินค้าแบบ Paginated พร้อมค้นหา/กรอง
getProductsForSelect()PRODUCT_VIEWรายการสั้นสำหรับ dropdown
getProductsForPurchase()PURCHASE_CREATEสินค้าสำหรับหน้าสร้างการซื้อ
getLowStockProducts()PRODUCT_VIEWสินค้าที่สต็อกต่ำกว่า minStock
createProduct()PRODUCT_CREATEสร้างสินค้าใหม่ ตั้งค่า isLowStock อัตโนมัติ
updateProduct()PRODUCT_EDITแก้ไขสินค้า (Optimistic Locking)
deleteProduct()PRODUCT_DELETESoft delete ตรวจสอบ active sales
adjustStock()STOCK_ADJUSTปรับสต็อก (atomic) + StockLog
batchCreateProducts()PRODUCT_CREATEนำเข้า CSV สร้างหลายรายการ

3. การขาย (Sales)

ไฟล์: src/actions/sales.ts

ฟังก์ชันสิทธิ์ผลข้างเคียง
createSale()SALE_CREATE / POS_ACCESSตัดสต็อก (atomic), สร้างเลขใบเสร็จ (race-safe), StockLog, แจ้งเตือน
getSales()SALE_VIEW—
getSaleById()SALE_VIEW—
cancelSale()SALE_CANCELคืนสต็อก (atomic), ยกเลิก shipment, StockLog
verifyPayment()PAYMENT_VERIFYอัปเดตสถานะการชำระ, บันทึกผู้ตรวจสอบ
uploadPaymentProof()SALE_VIEWอัปโหลดรูปสลิป, ตั้งสถานะ PENDING

Race Condition Safety: เลขใบเสร็จสร้างด้วย COUNT + retry loop ภายใน transaction ป้องกันเลขซ้ำในกรณี concurrent
